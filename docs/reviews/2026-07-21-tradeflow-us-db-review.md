## 3. 즉시 연결을 차단하는 항목

### 3.1 가격 조정 방식과 배당 이중 반영

생성 코드는 yfinance를 `auto_adjust=True`로 호출한다. 따라서 `daily_price` OHLC는 분할뿐 아니라 배당까지 반영된 조정가격이다. 반면 현재 구현 계약은 다음과 같다.

- 원본 OHLC와 split-adjusted OHLC를 모두 저장한다.
- 전략과 체결에는 split-adjusted 가격을 사용한다.
- `cash_dividend`를 포트폴리오 현금에 별도로 한 번 반영한다.

현재 DB의 조정가격을 split-adjusted 가격으로 넣고 `corporate_actions.dividend`도 넣으면 배당 효과가 두 번 반영된다. 원본 OHLC가 없어서 DB만으로 계약에 맞는 값을 신뢰성 있게 복원할 수도 없다.

**필수 조치:** `auto_adjust=False`로 원본 OHLC와 corporate actions를 다시 수집하고, split-only 조정가격을 별도로 계산·저장한다. 임시로 조정가격을 원본과 split-adjusted 양쪽에 복사하거나 배당을 0으로 만드는 방식은 승인용 결과에는 사용하지 않는다.

### 3.2 출처와 수집 시각 부재

DB에는 행별 `source`, `fetched_at`이 없고 수집 실행 로그 테이블도 없다. 파일 수정시각을 행별 출처 메타데이터로 대체할 수 없다. 생성 코드상 가격 공급자는 KIS가 아니라 yfinance이지만, 이 사실이 DB 안에는 보존되지 않는다.

**필수 조치:** canonical 적재 시 `source`, timezone-aware `fetched_at`, 수집 실행 ID를 저장하고 원본 데이터셋 해시 또는 원본 파일 체크섬을 실행 단위로 남긴다.

### 3.3 PIT 유니버스 부재

문서와 생성 스크립트에는 `index_membership`이 언급되지만 실제 DB에는 해당 테이블이 없다. 현재 76종목이 어느 기준일에 어떤 근거로 포함되었는지 알 수 없으므로 전체 과거에 같은 목록을 적용하면 유니버스 등급 C다.

**필수 조치:** 공개 편입·제외 이력, 출처 URL/스냅샷, 확인된 누락 구간을 포함해 등급 B를 구성한다. 확보 전에는 Phase 1 승인 결과가 아니라 등급 C 민감도 결과로만 표시한다.

### 3.4 데이터 신선도

검수 시각은 미국 동부 2026-07-21 04:31이고 2026-07-20 정규장은 완료되었지만 가격 최대일은 2026-07-17이다. 수집기의 yfinance `end`가 배타적이라는 점과 KST/미국시장 기준일 계산을 함께 고쳐야 한다.

## 4. 권장 수용 순서

1. 공급자에서 원본 OHLCV와 corporate actions를 재수집한다.
2. 원본과 split-only 조정가격을 분리하고 배당 현금흐름을 한 번만 반영한다.
3. 시장 컨텍스트를 주식 `DailyBar`와 분리하고 음수값·캘린더 정렬 정책을 구현한다.
4. 출처·수집시각·실행 ID를 포함하는 읽기 전용 adapter를 만든다.
5. 41개 반올림 사례에 대한 정밀도 정책을 고정 fixture로 테스트한다.
6. 유니버스 등급 B를 구성한 뒤 등급 C 결과와 함께 10년 백테스트를 수행한다.
