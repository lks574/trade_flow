## 1. 개요

한국투자증권(KIS) Open API 기반 **미국주식 일봉 스윙 자동매매 프로그램**.
미국 시장으로 시작하고, 검증 완료 후 한국 시장으로 확장한다.

핵심 원칙:

- **전략 = 순수 함수.** 백테스터와 라이브 실행기가 동일한 전략 코드를 호출한다. 검증한 코드가 그대로 실전에 돈다.
- **스케줄 실행형.** 상주 데몬·웹소켓 없음. 하루 1회 실행 후 종료.
- **fail-safe.** 상태가 불확실하면 거래하지 않고 사람을 호출한다.

| 언어/환경 | Python, 로컬(맥)/서버 상주 스케줄 실행 | |

| 주간 탐색 | 전체 활성 유니버스에서 최대 5개 후보를 찾고 기존 종목과 비교 | 적격 후보가 없으면 빈 결과 |
| 검증 | 백테스트 → 모의투자 최소 20거래일 → 실계좌 | §9 |

## 3. 아키텍처

하루 1회 cron/launchd가 `run.py`를 실행:

```
run.py
 ├─ 1. KIS 토큰 발급/갱신
 ├─ 2. 잔고·미체결 주문 조회 및 로컬 상태 조정
 ├─ 3. 일봉 데이터 업데이트·신선도 검사
 ├─ 4. 감정점수와 정규화된 시장 이벤트 수집·기록
 ├─ 5. strategy.signal(data) → 원시 목표비중        ← 순수 함수
 ├─ 6. 일일 모니터링 리포트 생성                   ← 비실행형 분석
 ├─ 7. 리스크 정책 적용 → 최종 목표비중 (§6)        ← 뉴스 리포트와 분리
 ├─ 8. 현재잔고 diff → 매도 주문 → 체결·취소 확인
 ├─ 9. 실제 주문 가능 현금으로 매수 수량 재계산·집행
 ├─ 10. SQLite에 주문/체결/포트폴리오 스냅샷 기록
 └─ 11. 텔레그램 결과 요약 전송
```

### 모듈 구성

| 모듈 | 역할 |
|---|---|
| `broker/` | KIS REST 클라이언트 (인증, 시세, 잔고, 주문). 모의↔실전은 base URL+앱키 교체. 기존 오픈소스 래퍼(python-kis 등) 우선 검토, 안 맞으면 직접 REST 호출 |
| `data/` | 일봉 수집·SQLite 캐시. 백테스트용 장기 과거 데이터는 yfinance 보조. 감정점수 일일 수집·저장 |
| `strategy/` | 순수 함수 `signal(data) -> {symbol: weight}`. 팩터 스코어링 (§4) |
| `backtest/` | pandas 벡터 백테스터 (프레임워크 없음). 수수료·슬리피지 반영. 라이브와 동일한 strategy 함수 호출 |
| `engine/` | 목표비중→주문 변환, 집행, 기록, 멱등성 |
| `risk/` | 주문 전 게이트 (§6) |
| `monitoring/` | 보유종목 일일 검토, 시장 이벤트 연결, 주간 후보·교체 비교. 목표 비중과 주문을 만들지 않음 |
| `notify/` | 텔레그램 전송 |

## 4. 전략 v1: 다중 팩터 스코어링 랭킹

참고 문서(노션 "매수 추천 종목 선정 기준")의 구조를 채택하되, 문서에 없던 **매도 룰**과
**검증 불가 팩터의 격리**를 추가한다.

### 4.1 매수 (진입)

매 실행일:

1. 유니버스 각 종목에 팩터 점수 계산: `score = Σ (가중치 × 팩터)`
   - **v1 팩터 (전부 가격 데이터만으로 계산 → 백테스트 가능):**
     - 60거래일 모멘텀 (수익률)
     - 골든크로스 여부 (SMA50 > SMA200)
     - RSI(14) < 50
     - MACD line > signal line
   - **섀도 팩터 (가중치 0, 기록만):** 뉴스 감정점수 (§4.3)
   - **확장 슬롯 (v1 제외):** 외부 ML 예측값 — 전방 검증으로 유효성이 입증되기 전에는 가중치를 주지 않는다
2. 모멘텀은 적격 종목 내 percentile rank(0~1), 나머지 팩터는 0/1로 계산한다.
   초기 가중치는 모멘텀 0.40, 추세 0.25, RSI 0.15, MACD 0.20이다.
3. 점수 상위 N개 **그리고 종가 > SMA200** 인 종목을 균등 비중으로 보유
4. 조건 통과 종목이 0개면 **전량 현금** (하락장 자동 회피)

### 4.2 매도 (청산) — 셋 중 하나라도 해당하면 다음 실행 때 전량 매도

1. 점수 순위권(N위) 이탈
2. 종가가 SMA200 아래로 이탈
3. 평균 체결단가 대비 -10% 손절 (리스크 게이트에서 집행)

### 4.3 감정점수 운영 방식

- 과거 데이터를 구하지 않는다. **지금부터 매일 수집·저장**하며 전방 검증한다.
- **초기 소스 후보: Alpha Vantage NEWS_SENTIMENT API.** 기사별 티커 감정점수와
  관련도를 저장하고, 노션 감정 레이블과의 매핑은 Phase 2에서 공식 필드 기준으로 확인한다.
- **수집 범위: 그날 기술 팩터 점수 상위 20개 후보만 조회.** 실제 요청 예산은
  Phase 2 착수 시 공식 호출 한도와 이용 조건을 확인해 설정한다.
  감정점수는 상위 후보 간 순위 조정용이므로 하위권 조회는 불필요하다.
- 검증 기간 동안 가중치 0의 **섀도 팩터**로 기록만 하고, "감정점수를 반영했다면 종목
  선정이 어떻게 달라졌고 성과가 어땠는지"를 일일 비교 리포트로 누적한다.
- **최소 60거래일** 전방 검증 후 데이터가 유효하다고 판단되면 가중치를 부여한다.
  (모의투자 기간 1개월과 별개 — 수집은 실계좌 전환 후에도 계속되므로 일정을 막지 않는다)
- `sentiment` 테이블에 source 컬럼을 두어, 소스가 부실하면 헤드라인+로컬 FinBERT
  스코어링을 추가 소스로 병행할 수 있다 (v1 제외).

### 4.4 위기 레짐 필터 (세계정세 반영)

지정학·정책 쇼크를 뉴스 해석이 아닌 **측정 가능한 시장 지표**로 반영한다.

- 역사적 근거: 지정학 이벤트의 S&P 낙폭 중간값은 약 -5~6%, 수 주~수개월 내 회복이
  대다수. 구조적 하락장이 되는 유일한 패턴은 **유가 급등 → 인플레이션 전이**형
  (1973 오일쇼크). 따라서 뉴스 기반 즉각 매도는 반영하지 않고, 위험 전이를 시장
  지표로 감지한다.
- **입력:** VIX 종가(1990~), WTI 유가 20일 변화율. yfinance로 수집(^VIX, CL=F).
- **룰 (초기값, 백테스트로 확정):** `VIX > 30 또는 WTI 20일 상승률 > 30%` 진입,
  3거래일 연속 거짓이면 해제. 발동 시 기본 동작은 **신규 매수 중단** (전략 청산·
  손절 매도는 허용). 주식 비중을 강제 축소하는 "노출 축소 정책"은 Phase 1
  백테스트에서 비교 평가만 하고, 승인 전에는 실주문에 적용하지 않는다.
- 걸프전·9/11·2008·2020·2022 구간을 포함한 30년 백테스트로 임계값을 검증한다.
  **백테스트 결과 필터가 성과를 해치면 제외하는 것까지 데이터로 결정한다.**
- 지정학 뉴스 요약·레짐 지표 현황은 일일 리포트에 표시한다. 뉴스는 `watch`, `reduce`,
  `exit` 수동 검토 의견의 근거가 될 수 있지만 `execution_authorized=false`를 유지하며,
  검증 불가능한 뉴스 해석이 목표 비중이나 주문을 만드는 일은 없다.

### 4.5 유니버스

- **메인 슬리브:** S&P100급 미국 대형주 (설정 파일의 종목 리스트)
- **고변동 슬리브:** 고변동성 종목 소수. **총 비중 상한 10%, 종목당 5%** 로 격리
  - 근거: 하루 1회 실행 구조에서는 밤새 갭 리스크를 장중 대응으로 막을 수 없으므로, 비중 상한이 유일한 실효 방어선이다
- 유니버스는 코드가 아닌 설정 파일이므로 교체·확장에 코드 변경 불필요

### 4.6 파라미터 (전부 설정 파일, 백테스트로 확정)

초기값 제안: N=3~5, 모멘텀 60일, SMA 50/200, RSI 14, 손절 -10%, 팩터 가중치는
백테스트 그리드로 결정. **이 숫자들은 제안값이며 Phase 1 백테스트가 확정한다.**

### 4.7 일일 모니터링과 주간 탐색

- 미국 정규장 완료 후 보유종목의 1·5·20일 수익률, 거래량 변화, 추세, 손절, 전략 순위를
  검토한다.
- 출처·시각·영향 범위·방향·심각도·신뢰도가 있는 기업·거시·지정학 이벤트만 사용한다.
- 일일 신규 후보는 엄격한 진입 기준을 통과한 최대 3개만 보고하고 기준 미달이면 추천하지 않는다.
- 주 마지막 완료 세션에는 전체 활성 유니버스에서 최대 5개를 찾고 기존 종목의 점수·순위·
  이벤트 위험과 비교한다.
- 새로운 후보가 있다는 이유만으로 기존 종목을 교체하지 않는다. 기존 종목 약화와 최소 점수
  차이가 함께 확인되어야 한다.
- 위험 레짐 중에는 후보 관찰은 가능하지만 신규 진입과 교체를 추천하지 않는다.
- 정확한 임계값, 입력 enum, JSON 출력 계약은 tech spec §0.2와 §3.8을 따른다.

## 5. 체결 모델 · 자금 관리

### 5.1 체결 모델 (백테스트-라이브 정합성의 핵심)

- **신호: 전일 종가 기준 계산. 체결: 당일 개장 직후 주문.**
- 백테스트도 동일하게 **"종가 신호 → 익일 시가 체결"** 을 가정한다. 종가 신호를
  같은 날 종가에 체결하는 백테스트는 미래 참조(look-ahead)이므로 금지.
- 주문 유형: **지정가** (직전가 대비 소폭 여유를 둔 가격). 시장가 갭 체결 방지.
- 미체결 처리: 실행 종료 시점까지 미체결이면 **취소**하고 기록. 다음 실행 때
  신호가 유효하면 다시 계산해서 주문한다 (이월 주문 없음 — 상태 단순화).
- 부분 체결: 체결된 만큼만 잔고로 인정하고 다음 실행 때 diff로 자연 보정.

### 5.2 환전 · 예수금

- 미국주식 매매는 **달러 예수금** 기준으로 동작한다.
- **환전은 자동화하지 않는다** (수동, 사용자 직접). 프로그램은 달러 예수금
  잔고만 읽고, 부족하면 주문 축소 + 텔레그램으로 알린다.
- 계좌의 통합증거금 사용 여부는 Phase 2 시작 전 확인 항목.

## 6. 리스크 관리 (실계좌 필수 요건, 축소 금지)

- 종목당 최대 비중 / 최소 정수 1주
- 고변동 슬리브 비중 상한 (§4.5)
- 일일 손실 한도 초과 시 매수 중지, 전략 청산·손절·비중 축소 매도만 허용 + 텔레그램 경보
- **kill switch:** 지정 파일이 존재하면 주문을 내지 않고 상태·경보만 기록
- **dry-run 모드:** 주문 API만 스킵하고 전 과정 기록. 실계좌 전환 전 관문
- fail-safe: 잔고/체결 확인 실패 등 상태 불확실 시 거래하지 않고 경보

## 7. 데이터 (SQLite 단일 파일)

- `prices` — 일봉 OHLCV
- `sentiment` — 일일 감정점수 (종목, 날짜, 점수, 소스)
- `orders` — 주문 기록 (실행 ID, 멱등성 키 포함)
- `fills` — 체결 기록
- `snapshots` — 일일 포트폴리오 스냅샷 (전방 검증·성과 분석용)
- `runs` — 실행 로그 (시작/종료, 결과, 오류)
- 초기 모니터링은 포지션·시장 이벤트 JSON을 입력으로 받고 일일·주간 JSON 리포트를 출력한다.
  이벤트와 리포트의 DB 영속화는 운영 보존 정책 확정 후 추가한다.

## 8. 에러 처리 · 운영

- **주문 멱등성:** 결정적 주문 의도 ID 기반. 실행 ID는 추적에만 사용하며, 재실행 시 이미 낸 주문의 상태를 조회
- 조회 API 일시 오류만 자동 재시도. 주문 API 타임아웃은 재주문 전에 브로커 주문 내역으로 접수 여부 확인
- 모든 실행의 전체 로그를 파일+DB에 보존 (감사 기록)
- **휴장일:** 미국 증시 캘린더로 휴장일이면 즉시 종료 (`pandas_market_calendars` 등 기존 라이브러리 사용)
- **중복 실행 방지:** 실행 시작 시 락 파일, 종료 시 해제
- **KIS 토큰 캐싱:** 공식 인증 응답의 만료 시각까지 파일에 캐시하고 재사용. 토큰
  재발급 제한을 고려해 매 실행마다 새로 발급하지 않는다
- **시크릿 관리:** 앱키·토큰·API 키는 `.env` 파일로 관리, `.gitignore` 필수.
  코드·DB·로그에 시크릿을 남기지 않는다

## 9. 검증 로드맵 (개발 순서와 동일)

1. **Phase 1 — 백테스터 + 전략:** 과거 10년+ 일봉으로 v1 팩터·파라미터 확정
   - **이벤트 스터디 리포트 포함:** 주요 전쟁·정책 이벤트 구간(걸프전, 9/11,
     이라크전, 2008, 2020 코로나, 2022 우크라이나, 2025 관세쇼크 등)마다 전략의
     낙폭·회복일수·레짐 필터 발동 여부를 자동 리포트
   - **생존 편향 대응:** 현재 시점의 대형주 리스트로 과거를 돌리면 성과가
     부풀려진다. 공개 편입·제외 이력으로 근사 복원한 point-in-time 유니버스
     (tech spec의 등급 B)를 기본으로 사용하고, 높은 거래비용 가정으로 생존
     편향을 대체하지 않는다 (상세 기준은 tech spec §7.1)
2. **Phase 2 — KIS 모의투자 자동매매 (최소 20거래일):**
   - **첫 작업: 모의투자 서버의 해외주식 지원 범위 확인** (주문 유형, 시세 지연
     등 실전과의 차이). 제약이 크면 dry-run 모드 중심으로 검증 방식 조정
   - 모의 서버로 전체 주문 흐름 검증
   - 감정점수 매일 수집 + 섀도 팩터 전방 검증
3. **Phase 3 — 실계좌:** dry-run 병행 → 소액 → 본 운용
4. **이후 — 한국시장 확장:** broker에 국내주식 주문 API 추가 + 유니버스 확장

### Phase 2 → 3 전환 합격 기준 (전부 충족 시에만 실계좌 진행)

- 20거래일 연속 무인 운영 중 **치명 오류(중복 주문, 잘못된 수량/종목 주문) 0건**
- 실행 실패가 텔레그램 또는 외부 dead-man 감시로 100% 탐지됨 (조용한 실패 0건)
- 동일 신호·설정·데이터에 대한 백테스터와 모의투자의 목표 종목·방향·수량 일치
- 실제 수량 차이는 정수 내림·현금 여유분·수수료·부분 체결·미체결 취소로 전부 설명되며, 설명 불가 차이 0건
- kill switch·dry-run·일일 손실 한도 각 1회 이상 실제 동작 확인

테스트: 전략 결정성·미래 참조·수량 계산·리스크 게이트·주문 멱등성·부분 체결·취소 실패·환경 격리 검증.
